# Contributing

Thanks for your interest in contributing.

## Scope

This repository is an academic data science project focused on football talent analysis. Contributions should keep the project transparent, reproducible, and interview-safe.

## How to Contribute

1. Fork the repository.
2. Create a feature branch from `main`.
3. Make focused changes (one logical change per commit when possible).
4. Run basic checks locally.
5. Open a Pull Request with a clear summary.

## Branch and Commit Conventions

- Branch naming examples:
  - `feature/add-data-dictionary`
  - `fix/scraper-timeout-handling`
  - `docs/readme-improvements`
- Commit messages should be concise and action-oriented.

## Pull Request Checklist

- [ ] Change is clearly described.
- [ ] README/docs updated if behavior changed.
- [ ] No sensitive data or private credentials added.
- [ ] Changes are consistent with project scope.
- [ ] If code was added, it is runnable in the documented setup.

## Data and Compliance Notes

- Do not add copyrighted datasets or protected assets.
- Keep large raw datasets out of git unless intentionally versioned.
- Respect website terms when scraping and avoid abusive request patterns.

## Questions

Open an issue describing your proposal before major refactors.
